# ReproBranches
A collection of minimal bug reproduction cases, organized by branches.

## App
The app has two buttons, *First* and *Second*, each with a `CheckBox` next to it.
The `CheckBox` is disabled while the command is running, using `InvertedBoolConverter` on `IsRunning`.

The *First* button uses a standard compiled binding path from the ViewModel:
```xml
<CheckBox IsEnabled="{Binding FirstCommand.IsRunning, Converter={StaticResource InvertedBoolConverter}}" />
```
The *Second* button uses `x:DataType` with the full CLR name `CommunityToolkit.Mvvm.Input.IAsyncRelayCommand` inline on the binding:
```xml
<CheckBox IsEnabled="{Binding SecondCommand.IsRunning, Converter={StaticResource InvertedBoolConverter}, x:DataType=CommunityToolkit.Mvvm.Input.IAsyncRelayCommand}" />
```

When XAML source generation is enabled (`<MauiXamlInflator>SourceGen</MauiXamlInflator>`), the *First* button's `CheckBox` binding produces a compiler warning:

```
MainPage.xaml(22,28): Warning MAUIG2045 : Binding: Property "IsRunning" not found on "global::System.Windows.Input.ICommand". The binding will use slower reflection-based binding at runtime. If this property is generated by another source generator, consider reporting this at https://github.com/dotnet/maui/issues to help us improve source generator interoperability.
```
